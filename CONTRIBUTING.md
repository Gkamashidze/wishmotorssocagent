# Contributing to wishmotorssocagent

## Workflow

1. **Fork** the repository
2. **Create a branch:** `git checkout -b feature/your-feature-name`
3. **Make your changes** — follow the rules in `.claude/rules/`
4. **Run tests** — all must pass before committing
5. **Commit** using conventional format: `feat: add login form`
6. **Open a Pull Request** — fill in the template

## Commit Format
```
type: short description

[optional body]
```

Types: `feat`, `fix`, `docs`, `test`, `refactor`, `chore`, `perf`, `ci`

## Before Submitting a PR
- [ ] Tests pass
- [ ] Linter passes (zero errors)
- [ ] No secrets in code
- [ ] Documentation updated if needed

## Questions?
Open an issue with the `question` label.
